HTML 디자인/구조 가이드

1편 - 필수 용어와 공통 구조

왜 용어를 전부 외우지 않을까?

HTML 개발에는 수백 개의 용어가 있지만, 실제로 반복해서 사용하는 것은 많지 않습니다. 먼저 아래 15개 정도만 이해하면 Claude, ChatGPT, Gemini와 협업할 때 대부분의 대화를 이해할 수 있습니다.

필수 용어

용어역할쉽게 이해하기
HTML화면의 구조건물의 뼈대
CSS색상·폰트·레이아웃 등 디자인인테리어
JavaScript버튼, 팝업, 기능전기 설비
JSON설정·데이터 저장창고 목록
API다른 서비스와 통신택배 기사
fetch()API 호출전화 걸기
GAS구글 자동화 서버회사 직원
GitHub프로젝트 보관클라우드 창고
Commit변경사항 저장게임 저장
Deploy웹사이트 공개오픈
Function기능 묶음기계 하나
Variable값 저장상자
Array목록 저장서랍
Object이름+값 저장사원 카드
Cache임시 저장책상 위 메모

여러 HTML을 만들 때의 정석 구조

project/ ├── html/ ├── css/ │ ├── common.css │ ├── dashboard.css │ └── theme.css ├── js/ │ ├── common.js │ ├── api.js │ └── utils.js ├── json/ │ ├── config.json │ └── data.json └── assets/

공통 파일의 역할

파일담아둘 내용
common.css버튼, 카드, 폰트, 테이블, 입력창, 색상 등 모든 페이지 공통 디자인
common.jsToast, Loading, Modal, 날짜 포맷, 복사 기능 등 공통 기능
api.jsGAS, Notion API, GitHub API 통신
config.jsonAPI URL, 버전, 회사명 등 설정값
data.json실시간이 아닌 대용량 데이터

현재 프로젝트에 가장 적합한 운영 방식

  • 실시간 데이터 → GAS + API
  • 대용량 데이터 → JSON
  • 공통 디자인 → common.css
  • 공통 기능 → common.js
  • 설정값 → config.json
  • GitHub에서 전체 프로젝트 관리

핵심: HTML마다 디자인를 따로 만들지 말고, 공통 CSS와 공통 JS를 사용하면 유지보수가 훨씬 쉬워진다.
