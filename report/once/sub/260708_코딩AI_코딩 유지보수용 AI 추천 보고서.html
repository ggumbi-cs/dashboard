다만 전부 유료로 시작할 필요는 없습니다. 먼저 Cursor 또는 Copilot 중 하나를 골라 유지보수 체감을 보고, GAS 쪽 문제가 반복될 때 Gemini를 붙이는 방식이 좋습니다.

개인적으로는 GitHub 저장소 중심으로 관리한다면 Copilot/Codex 먼저, 로컬 HTML 파일을 자주 직접 고친다면 Cursor 먼저가 더 효율적입니다.

확인한 근거

  • GitHub Copilot은 여러 AI 모델을 지원하고, 모델별 강점에 따라 선택할 수 있다고 안내합니다. GitHub Docs
  • GitHub는 OpenAI Codex coding agent를 유료 Copilot 플랜에서 사용할 수 있다고 설명합니다. GitHub Docs
  • GitHub는 외부 coding agent가 issue나 prompt를 받아 작업 후 pull request를 만들 수 있는 흐름을 설명합니다. GitHub Docs
  • Google Apps Script는 Google 제품을 통합·자동화하는 클라우드 기반 JavaScript 플랫폼입니다. Google Developers
  • Apps Script API는 스크립트 생성, 수정, 배포, 원격 실행을 자동화할 수 있다고 안내합니다. Google Developers
  • GitHub REST API는 통합 생성, 데이터 검색, 워크플로 자동화에 사용할 수 있습니다. GitHub Docs
  • Gemini Code Assist는 IDE에서 코드 생성, 코드 변환, 코드 완성 등을 지원합니다. Google Developers
  • Google은 2026년 6월 18일부터 개인용 Gemini Code Assist IDE 확장 및 Gemini CLI 요청 지원 중단을 공지했습니다. Google Developers
