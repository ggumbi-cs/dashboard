말할 내용
안녕하세요, 엄마감동팀 심우승입니다. 오늘은 저희 팀이 CS와 품질관리 업무를 하면서 AI를 활용해 직접 만든 업무 시스템을 소개하려고 합니다. (잠시 멈춤) 거창한 프로젝트로 시작한 게 아니라, 매일 반복되던 작업 하나를 자동화하다 보니 지금의 구조까지 오게 됐는데요, 그 과정을 순서대로 보여드리겠습니다.
포인트
  • "거창하게 시작한 게 아니다"를 강조 — 전직원 대상이므로 진입장벽을 낮추는 게 목적
  • 발표자료 자체도 이 시스템과 같은 방식(HTML)으로 만들었다는 점은 마무리에서 다시 언급 예정
01
문제의식 — Before/After
1:00–3:00
말할 내용
예전에는 CS·품질 데이터가 여러 시트와 문서에 흩어져 있었고, 보고서도 매번 새로 정리해야 했습니다. 부품 원인 분석이나 견적 산출도 결국 경험 많은 사람의 감에 의존했었죠. (전환) 지금은 하나의 대시보드에서 데이터가 통합되고, 보고서는 정해진 양식으로 자동 집계되고, 과거 데이터를 기반으로 원인을 자동 추천해주는 구조로 바뀌었습니다.
포인트
  • 수치(몇 % 감소 등)는 절대 언급하지 않기 — 근거 없는 수치 금지 원칙
  • "감에서 데이터로" 이 한 문장이 핵심 메시지
02
시스템 구조 — 4단계 아키텍처
3:00–7:00
말할 내용
구조는 크게 4단계입니다. 첫째, 데이터는 노션·채널톡처럼 API 연동이 되는 곳은 직접 연결하고, 안 되는 곳은 CSV로 내려받아 폴더에 넣습니다. 둘째, 이 데이터들은 전부 구글 앱스 스크립트(GAS)라는 창구를 거쳐 주고받는데요, 무거운 데이터는 매일 새벽에 미리 JSON으로 굳혀놔서 로딩이 빠릅니다. 셋째, 사용자는 역할에 따라 접근 권한이 다른 허브 화면 하나로 모든 걸 이용합니다. 넷째, 이 구조 자체를 설명하는 매뉴얼 페이지가 따로 있어서, 시스템이 스스로를 문서화합니다.
발표 팁
  • 이 파트가 가장 "기술적으로 보일 위험"이 있는 구간 — 천천히, 비유 위주로 설명
  • "용어에 마우스를 올리면 설명이 뜬다"는 점을 화면 보여주며 짧게 시연 (질문 나올 걸 대비한 장치라고 언급하면 좋음)
Q. GAS가 뭔가요? / 서버 비용은요?
A. 구글이 무료로 제공하는 스크립트 실행 환경입니다. 별도 서버 비용 없이 구글 계정 권한만으로 데이터를 주고받을 수 있어요.
03
구축 여정 — 타임라인
7:00–9:30
말할 내용
처음엔 주간·월간 보고서를 표준화하는 것부터 시작했습니다. 그다음 품질 지표를 세분화했고, 반복되는 실무 판단(부품 원인 추천, 견적 계산 등)을 자동화한 Tool을 12개까지 늘렸어요. 이후엔 팀원들이 직접 만들 수 있도록 13회차 교육을 진행했고, 지금은 저희 팀을 넘어 전사 지식공유 프로그램까지 이 방식으로 운영하고 있습니다.
포인트
  • "5단계"를 손가락으로 하나씩 짚듯이 리듬감 있게 — 이 파트가 스토리의 중심축
04
라이브 데모 — 실제 화면 전환
9:30–15:00
말할 내용 (전환 멘트)
설명은 이 정도로 하고, 실제 화면으로 직접 보여드리겠습니다. (카드 클릭 → 새 창으로 대시보드 오픈)
클릭 순서 · 화면별 멘트
  • ① AS1→AS3 연동: "모델이랑 증상만 고르면, 과거 수리 이력 기반으로 원인 부품이 랭킹으로 뜨고 — 이게 옆 화면에 실시간으로 넘어가서 관리방법까지 자동으로 추천돼요."
  • ② 자재발주 현황판: "상태 하나만 바꿔도 바로 반영되고, 기안일 기준 경과일도 색상으로 자동 표시됩니다."
  • ③ 통합 마감전송 허브: "9개 업무 항목을 한 화면에서 관리하고, 주간/월간 단위로 체크해서 한 번에 보고서 데이터로 넘깁니다."
  • ④ 지식DB: "이게 팀 업무가 아니라 전사 지식공유 프로그램인데, 같은 방식으로 조편성부터 미제출자 현황까지 관리하고 있어요."
⚠ 사전 체크 발표 PC 팝업 차단 해제 / 대시보드 로그인 세션 유지 / 인터넷 연결 상태 확인
05
팀 · 조직 확산
15:00–17:00
말할 내용
이걸 저 혼자만 쓰면 의미가 크지 않다고 생각해서, 팀원들이 직접 자기 업무 Tool을 만들 수 있도록 13회차 교육을 진행했습니다. "AI로 AI 활용하기"부터 시작해서 마지막엔 GAS로 자동 동기화까지 실습으로 다뤘고요. 지금은 그 경험이 팀을 넘어 전사 지식공유 프로그램에도 적용되고 있습니다.
포인트
  • "확산"이라는 단어를 명확히 강조 — 개인기가 아니라 조직 역량으로 이어졌다는 메시지
06
이 시스템이 만든 변화
17:00–18:30
말할 내용
숫자로 "몇 % 줄었다"는 말씀은 일부러 안 드립니다. 근거 없는 수치는 오히려 신뢰를 깎는다고 생각해서요. 대신 확실히 말씀드릴 수 있는 건, 보고서가 표준화됐고, 개인의 경험이 팀 전체가 쓸 수 있는 데이터 자산이 됐고, 팀원들이 이제 직접 도구를 만들 수 있게 됐다는 점입니다.
Q. 그래서 시간이 얼마나 줄었나요?
A. 정확한 근거 없이 수치로 말씀드리긴 어렵고, 대신 반복 작업이 자동화되면서 판단과 대응에 쓰는 시간이 늘었다는 정성적 변화로 봐주시면 좋겠습니다.
07
향후 계획 & 클로징
18:30–20:00
말할 내용
앞으로는 노션 API를 직접 연동하고, 깃허브 액션으로 배포 자동화까지 붙일 계획입니다. 스마트스토어 연동 문제도 검증 중이고요. (마무리) 오늘 보여드린 발표 자료 이 페이지 자체도, 저희 팀이 쓰는 방식 그대로 HTML로 만든 겁니다. 특별한 도구가 아니라, 필요한 걸 하나씩 만들다 보면 이렇게 된다는 말씀을 드리고 싶었습니다. 감사합니다.
포인트
  • 마지막 문장은 "이 발표자료도 같은 방식으로 만들었다"는 메타 포인트로 임팩트 있게 마무리
