AI에게 무엇을 요청하나요?
복사해서 바로 보내세요
💬 초보자용 짧고 편하게
현재 사용 중인 GAS 코드에 매일 오후 1시경 자동으로 데이터를 동기화하는 기능을 추가하고 싶어요. 자동 실행할 함수도 만들어주시고, 처음 사용하는 사람도 따라 할 수 있게 트리거 설정 방법을 순서대로 알려주세요. 처음에 수동으로 한 번 실행해야 하는 함수가 있다면 그 함수 이름과 이유도 알려주세요.
⚡ 추천 버전 더 구체적으로
기존 Google Apps Script 프로젝트에 매일 13시대 자동 동기화 기능을 추가해주세요. 동기화 본체 함수와 시간 기반 트리거 생성 함수를 구분해서 작성해주세요. 최초 설정 시 사용자가 수동으로 1회 실행해야 할 함수명을 명확히 표시해주세요. 권한 승인, 트리거 중복 생성 방지, 실행 실패 확인 방법도 포함해주세요. 기존 웹앱 배포 URL은 변경하지 않고 사용할 수 있도록 구성해주세요. 초보자가 따라 할 수 있는 등록 순서도 함께 설명해주세요.
4
AI 답변에서 딱 두 가지만 찾으세요
함수 이름 두 개만 확인하면 됩니다
① 실제 동기화 함수
syncData
실제로 데이터를 읽고 Google Sheet를 갱신하는 함수예요. 트리거가 매일 이 함수를 자동으로 불러요.
② 트리거 설치 함수
setupDailyTrigger
매일 오후 1시경 syncData가 실행되도록 예약을 등록하는 함수예요. 딱 한 번만 실행하면 돼요.
⚠ 함수 이름은 AI가 작성한 코드에 따라 달라질 수 있어요.
AI가 "이 함수를 최초 한 번 실행하세요"라고 지정한 함수명을 확인하세요.

❓ 자동화인데 왜 처음에는 직접 실행하나요?
최초 실행 시 Google이 확인해야 할 사항이 있어요.
· 이 코드가 시트를 수정해도 되는지
· 외부 데이터를 불러와도 되는지
· 사용자가 이 자동화를 허용했는지

그래서 트리거 설정 전에 AI가 지정한 함수를 딱 한 번만 수동으로 실행해요.

방식 A — 트리거 자동 등록
setupDailyTrigger 같은 설치 함수를 한 번 실행하면 자동으로 매일 실행 예약이 생성돼요.
방식 B — 직접 등록
Apps Script 트리거 메뉴에서 직접 예약을 만들고 실행할 함수로 syncData를 선택해요.
💡 AI가 준 설명에서 "자동 등록 방식"인지 "직접 등록 방식"인지 먼저 확인하세요.
한 번 실행하는 것은 매일 직접 실행하라는 뜻이 아닙니다.
처음 문을 열어주는 과정이에요. 이후에는 트리거가 자동으로 실행해요.

10
완료 후 AI에게 알려주기
상황에 맞는 문장을 복사해서 보내세요
✅ 정상 등록된 경우
GAS 코드 등록과 최초 실행을 완료했고, 매일 오후 1시~2시 시간 기반 트리거도 생성했습니다. 이제 HTML이나 기존 코드에서 추가로 수정할 부분이 있는지 확인해주세요.
❌ 실행 오류가 난 경우
GAS 함수를 수동 실행했는데 오류가 발생했습니다. 오류 화면과 실행 로그를 함께 첨부할게요. 어느 부분을 수정해야 하는지 초보자도 이해할 수 있게 알려주세요.
⚠ 트리거가 실행되지 않는 경우
시간 기반 트리거는 등록되어 있지만 데이터가 갱신되지 않았습니다. 트리거 설정 화면과 실행 기록을 첨부할게요. 함수명과 권한, 실행 오류를 확인해주세요.
